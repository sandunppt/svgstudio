# Contributing

感谢你愿意参与本项目。

## 开发环境

1. Fork 并克隆仓库
2. 安装依赖

```bash
npm install
```

3. 启动开发服务

```bash
npm run dev
```

默认地址：`http://localhost:5999`

## 分支规范

- 新功能：`feature/<short-name>`
- 修复问题：`fix/<short-name>`
- 文档优化：`docs/<short-name>`
- 重构：`refactor/<short-name>`

## 提交规范（建议）

使用清晰的前缀：

- `feat:` 新功能
- `fix:` 缺陷修复
- `docs:` 文档更新
- `refactor:` 代码重构
- `style:` 样式调整
- `chore:` 构建/配置等杂项

示例：

```text
feat: add reset button beside undo
fix: normalize text font size after scaling
docs: update README for open-source release
```

## Pull Request 要求

提交 PR 前请确认：

1. 代码能正常运行，主要功能无回归
2. 改动范围尽量聚焦，避免混入无关修改
3. PR 描述里写清：
   - 改了什么
   - 为什么改
   - 如何验证
4. 如果涉及 UI，建议附截图或录屏

## Issue 建议信息

提 Issue 时建议提供：

- 复现步骤
- 实际结果 / 预期结果
- 浏览器和系统信息
- 控制台报错（如有）

## 行为准则

请保持尊重、客观、建设性的沟通方式。
